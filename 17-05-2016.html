###### 17-05-2016 # This week in Parity #1 Hello and welcome to first issue of *This week in Parity*! Parity is world's fastest and lightest Ethereum client. Written in the sophisticated, cutting-edge Rust language. This is a weekly summary of its progress and community. ## What's new in Parity? - [@arkpar](https://github.com/arkpar) was polishing the new syncing code and bugfixing. - [@debris](https://github.com/debris) finished working on [ethabi](https://github.com/ethcore/ethabi) - library and command line tool to interact with evm contracts. - [@gavofyork](https://github.com/gavofyork) was bugfixing rpc api. - [@NikVolf](https://github.com/NikVolf) added configuration for keys security, multiple refactoring to get database api ready for migration to ipc architecture. - [@tomusdrw](https://github.com/tomusdrw) has been working on domain-based routing in webapps when using parity as a proxy (with proxy.pac). Identity management topbar for dapps. - [@General-Beck](https://github.com/General-Beck) was busy working on cross platform travis builds, mining, travis, mining and even more travis and mining ;) - ... and many other small things. ## What's next? - Fat DB [#138](https://github.com/ethcore/parity/issues/138) - State-sync "PV63" [#149](https://github.com/ethcore/parity/issues/149) - Windows build [#462](https://github.com/ethcore/parity/issues/462) - Optimise gas calculation in EVM [#491](https://github.com/ethcore/parity/issues/491) - Split network, JSONRPC, rocksdb & core into separate processes [#492](https://github.com/ethcore/parity/issues/492) - CLI option for using JITEVM [#619](https://github.com/ethcore/parity/issues/619) - Refactor syncing code to use requests by hashes rather than by numbers [#696](https://github.com/ethcore/parity/issues/696) - Stratum/Eth API for mining [#755](https://github.com/ethcore/parity/issues/755) - RESTful interface module [#765](https://github.com/ethcore/parity/issues/765) - Comprehensive tests for client module [#863](https://github.com/ethcore/parity/issues/863) ## List of all contributors so far - [@arkpar](https://github.com/arkpar) - [@debris](https://github.com/debris) - [@gavofyork](https://github.com/gavofyork) - [@General-Beck](https://github.com/General-Beck) - [@goldylucks](https://github.com/goldylucks) - [@jesuscript](https://github.com/jesuscript) - [@kobigurk](https://github.com/kobigurk) - [@loxal](https://github.com/loxal) - [@Manishearth](https://github.com/Manishearth) - [@NikVolf](https://github.com/NikVolf) - [@peterjoel](https://github.com/peterjoel) - [@petevine](https://github.com/petevine) - [@poppingtonic](https://github.com/poppingtonic) - [@guanqun](https://github.com/guanqun) - [@rphmeier](https://github.com/rphmeier) - [@tomusdrw](https://github.com/tomusdrw) - [@wlk](https://github.com/wlk) ## Upcoming Events Between May 22 - June 3 our team will be in Berlin.
