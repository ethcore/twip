###### 25-05-2016 # This week in Parity #2 Hello and welcome to second issue of *This week in Parity*! Parity is world's fastest and lightest Ethereum client. Written in the sophisticated, cutting-edge Rust language. This is a weekly summary of its progress and community. ## What's new in Parity? - [@arkpar](https://github.com/arkpar) finished new syncing strategy (using hashes instead of block numbers) [#1095](https://github.com/ethcore/parity/pull/1095) and has been working on the **Windows build** - it's almost done! - [@gavofyork](https://github.com/gavofyork) coded blockchain importing/exporting ([#1129](https://github.com/ethcore/parity/pull/1129)/[#1132](https://github.com/ethcore/parity/pull/1132)), coloured & padded logging [#1133](https://github.com/ethcore/parity/pull/1133), enhanced CLI options for controlling key importing (`--no-import-keys`, `--testnet`) [#1119](https://github.com/ethcore/parity/pull/1119) and fixed the seal fields in RPC output [#1096](https://github.com/ethcore/parity/pull/1096) and the RPC `is_syncing` method [#1122](https://github.com/ethcore/parity/pull/1122). - [@tomusdrw](https://github.com/tomusdrw) enhanced the Parity Webapps API, added the Home Webapp [#1101](https://github.com/ethcore/parity/pull/1101) and ported DAO and Maker OTC Dapps to Parity [#1113](https://github.com/ethcore/parity/pull/1113). - [@debris](https://github.com/debris) added CLI option for using JIT EVM [#1103](https://github.com/ethcore/parity/pull/1103), did foundation work for database migration [#1128](https://github.com/ethcore/parity/pull/1128), did the first such migration [#1014](https://github.com/ethcore/parity/pull/1014) and improved naming consistency across the codebase [#1105](https://github.com/ethcore/parity/pull/1105). - [@NikVolf](https://github.com/NikVolf) refactored the miner to hold its own copy of spec and engine [#1091](https://github.com/ethcore/parity/pull/1091) and has been working on separating database layer into it's own crate and process [#1097](https://github.com/ethcore/parity/pull/1097), [#1115](https://github.com/ethcore/parity/pull/1115) - [@rphmeier](https://github.com/rphmeier) implemented error pretty printing [#1116](https://github.com/ethcore/parity/pull/1116) - [@jacogr](https://github.com/jacogr) has been working on JavaScript port of [ethabi](https://github.com/ethcore/ethabi) - [ethabi-js](https://github.com/jacogr/ethabi-js) - ... and many other small things. ## What's next? - Fat DB [#138](https://github.com/ethcore/parity/issues/138) - RESTful interface module [#765](https://github.com/ethcore/parity/issues/765) -
