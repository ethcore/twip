###### 17-05-2016 # This week in Parity #1 Hello and welcome to first issue of *This week in Parity*! Parity is world's fastest and lightest Ethereum client. Written in the sophisticated, cutting-edge Rust language. This is a weekly summary of its progress and community. ## What's new in Parity? - [@gavofyork](https://github.com/gavofyork) added geth IPC compatibility [#1059](https://github.com/ethcore/parity/pull/1059) - [@arkpar](https://github.com/arkpar) added transaction signature validation in tx queue [#1068](https://github.com/ethcore/parity/pull/1068) - [@arkpar](https://github.com/arkpar) fixed connecting to local nodes on startup [#1070](https://github.com/ethcore/parity/pull/1070) - [@NikVolf](https://github.com/NikVolf) added configuration for security keys [#1080](https://github.com/ethcore/parity/pull/1080) - [@NikVolf](https://github.com/NikVolf) improved configuration of database directories across platforms [#1083](https://github.com/ethcore/parity/pull/1083) - [@gavofyork](https://github.com/gavofyork) fixed rpc bug in eth_getUncleByBlockNumberAndIndex [#1087](https://github.com/ethcore/parity/pull/1087) - [@tomusdrw](https://github.com/tomusdrw) has been working on domain-based routing in webapps when using parity as a proxy (with proxy.pac) [#1082](https://github.com/ethcore/parity/pull/1082) - [@arkpar](https://github.com/arkpar) implemented syncing block collections [#1088](https://github.com/ethcore/parity/pull/1088) - [@tomusdrw](https://github.com/tomusdrw) implemented identity management topbar for dapps [#1092](https://github.com/ethcore/parity/pull/1092) - [@debris](https://github.com/debris) fixed incorrect decoding of header seal_fields [#1094](https://github.com/ethcore/parity/pull/1094) - [@NikVolf](https://github.com/NikVolf) has been refactoring database api to get it ready for migration to ipc architecture. - [@General-Beck](https://github.com/General-Beck) was busy working on cross platform travis builds, mining, travis, mining and even more travis and mining ;) - [@debris](https://github.com/debris) finished working on [ethabi](https://github.com/ethcore/ethabi) - library and command line tool to interact with evm contracts - ... and many other small things. ## What's next? - Fat DB [#138](https://github.com/ethcore/parity/issues/138) - State-sync "PV63" [#149](https://github.com/ethcore/parity/issues/149) -
